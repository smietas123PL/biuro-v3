Ocena po poprawkach: Solidna Beta — gotowa do wewnętrznego deploymentu
Trzy commity rozwiązały 4 z 5 krytycznych issues z poprzedniego audytu. System agentowy jest teraz funkcjonalny end-to-end dla wszystkich trzech providerów LLM. API keys są szyfrowane AES-256-GCM, auth middleware blokuje nieautoryzowany dostęp, helm chroni nagłówki, mathjs zastępuje ryzykowny vm sandbox, a limit głębokości delegacji eliminuje ryzyko nieskończonych pętli. To naprawdę duży skok jakościowy w jednym sprincie. Pozostają 3 istotne kwestie: klucz szyfrowania ENCRYPTION_KEY ma fallback na hardcoded string, CORS nadal zezwala na wszystkie originy (origin: true), a CI pipeline nadal nie uruchamia testów. Przed zewnętrznym deployment — konieczne.

Naprawione krytyczne issues

4/5 kryt. → zamknięte
✓ ROZWIĄZANE LLM stuby — OpenAI i Anthropic używają prawdziwych SDK
Zainstalowano openai ^6.32.0 i @anthropic-ai/sdk ^0.80.0. Obie funkcje teraz tworzą klienta, wysyłają prompt z rolą agenta i zwracają rzeczywistą odpowiedź. Koszt nadal hardcoded $0.01 (patrz: otwarte issues).
✓ ROZWIĄZANE Szyfrowanie API keys — AES-256-GCM
Nowy plik utils/encryption.js z encrypt() i decrypt(). Klucz szyfrowany przed zapisem do bazy, odszyfrowany przy wywołaniu LLM. Format: iv:authTag:encrypted. Implementacja poprawna — GCM z losowym IV i auth tag.
✓ ROZWIĄZANE Auth middleware — X-API-Key na wszystkich endpointach
Middleware sprawdza header X-API-Key na każdym żądaniu. Pomija /api/health i środowisko test. Frontend w main.jsx dodaje nagłówek automatycznie przez fetch interceptor.
✓ ROZWIĄZANE vm.Script → mathjs.evaluate()
Narzędzie calculate w tools.js używa teraz evaluate() z mathjs — bezpieczny parser matematyczny bez możliwości ucieczki z sandboxa. Poprawna naprawa.
✓ ROZWIĄZANE Stub endpointy API — 10+ uzupełnionych implementacji
Uzupełnione: PATCH /agents/:id/budget, PATCH /agents/:id/active, PATCH /agents/:id/parent, GET|POST|DELETE /agents/:id/tools, PATCH /tasks/:id/done, PATCH /tasks/:id/priority, PATCH /tasks/:id/archive, PATCH /tasks/:id/unarchive, DELETE /agents/:id (z kaskadą). Wszystkie zapisują do bazy.
✓ ROZWIĄZANE MAX_DEPTH = 5 dla delegacji i FEEDBACK loop
Kolumna depth INTEGER DEFAULT 0 w tabeli tasks. Heartbeat przekazuje i inkrementuje głębokość przy każdej delegacji i pętli feedbacku. Zapobiega runaway costs.

Pozostałe otwarte issues

3 istotne · 5 drobnych
✗ Istotny ENCRYPTION_KEY ma fallback na hardcoded string
W utils/encryption.js:
const KEY = process.env.ENCRYPTION_KEY || 'default-long-secret-key-32-chars-!!'; // Jeśli ENCRYPTION_KEY nie jest ustawiony w .env, // wszystkie klucze szyfrowane są TYM SAMYM kluczem publicznym. // Osoba znająca kod może deszyfrować bazę danych każdego deploymentu. // Fix: rzuć wyjątek jeśli klucz nie istnieje: if (!process.env.ENCRYPTION_KEY) throw new Error('ENCRYPTION_KEY env var required'); const KEY = process.env.ENCRYPTION_KEY.slice(0, 32).padEnd(32);
✗ Istotny CORS origin: true — nadal zezwala na wszystkie domeny
Poprzednie app.use(cors()) zastąpione przez cors({ origin: true, ... }), co jest funkcjonalnie identyczne — każda domena ma dostęp. Komentarz w kodzie: "Allow all origins for now to fix 'Failed to fetch'". Właściwy fix: origin: process.env.FRONTEND_URL || 'http://localhost:5173'.
✗ Istotny CI pipeline nadal nie uruchamia testów
Workflow docker-build.yml bez zmian — tylko docker compose build. Przy 39 plikach testów, brak ich uruchomienia w CI to zmarnowany potencjał i ryzyko regresji.
# Dodać do docker-build.yml: - name: Run backend tests run: docker compose run --rm backend npm test - name: Run frontend tests run: docker compose run --rm frontend npm test
⚠ Uwaga Auth middleware używa domyślnego klucza 'biuro-secret-key'
const expectedKey = process.env.APP_API_KEY || 'biuro-secret-key' — analogicznie do ENCRYPTION_KEY, jeśli env nie jest ustawiony, klucz auth jest publicznie znany z kodu źródłowego. Ten sam problem co z kluczem szyfrowania.
⚠ Uwaga Koszt LLM nadal hardcoded $0.01 dla OpenAI i Anthropic
Nowe implementacje callOpenAI() i callAnthropic() zwracają calcCost = 0.01 niezależnie od liczby tokenów i modelu. GPT-4o kosztuje inaczej niż GPT-3.5. Budget tracking w bazie jest fałszywy. Anthropic API zwraca response.usage z input/output tokens, OpenAI podobnie — warto to wykorzystać.
⚠ Uwaga CSP wyłączone: contentSecurityPolicy: false
Helmet dodany — świetnie. Ale Content Security Policy wyłączone: helmet({ contentSecurityPolicy: false }). CSP to kluczowa ochrona przed XSS, szczególnie ważna bo API keys są w localStorage frontendowym. Warto skonfigurować minimalne CSP zamiast wyłączać całkowicie.
ℹ Info PATCH /tasks/:id/schedule nadal stub
Planowanie zadań (scheduled_for) nadal zwraca tylko { success: true } bez zapisu. Kolumna istnieje w schemacie, heartbeat respektuje ją w zapytaniu — endpoint jest ostatnim brakującym ogniwem.
ℹ Info Fetch interceptor w main.jsx — ryzyko circular dependency przy testach
Nadpisanie window.fetch globalnie w main.jsx może powodować niespodziewane zachowanie w testach Vitest (jsdom environment). Lepszym wzorcem jest axios z interceptorem lub dedykowany moduł api.js z wstrzyknięciem nagłówka.
ℹ Info Brak walidacji Zod na pozostałych endpointach
Zod dodany tylko dla POST /agents i POST /tasks. Endpointy PATCH (np. /agents/:id/budget przyjmuje req.body.budget bez walidacji czy to liczba) mogą dostać NaN lub string i zapisać do bazy bez błędu.
ℹ Info Template apply nadal hardkoduje companyId = 1
Niezmienione od poprzedniego audytu: const companyId = 1; // Default. Przy multi-company usage szablon zawsze nadpisuje dane pierwszej firmy, ignorując CompanySelector.

Nowe dobre wzorce wprowadzone w tym sprincie

✓ AES-256-GCM z losowym IV i auth tag — poprawna kryptografia
Implementacja w encryption.js jest poprawna: 12-bajtowy losowy IV, auth tag weryfikowany przy dekrypcji, obsługa błędów dekrypcji. Jedyna luka to fallback klucza (patrz: otwarte issues).
✓ GET /agents maskuje api_key jako '***'
agents.map(a => ({ ...a, api_key: a.api_key ? '***' : null })) — klucze nigdy nie wychodzą przez API, nawet zaszyfrowane. Dobra praktyka defensywna.
✓ Fetch interceptor — transparentne dodawanie auth header do wszystkich requestów
Eleganckie rozwiązanie — cały frontend działa bez zmian, a każde żądanie automatycznie dostaje X-API-Key. Wzorzec znany z axios interceptors.
✓ Accessibility: label + htmlFor + id na wszystkich polach formularzy
AddAgentModal, ApiKeysPanel, TaskList — wszystkie pola mają poprawnie powiązane etykiety. Dobra praktyka WCAG, ważna też dla automatycznych testów accessibility.
